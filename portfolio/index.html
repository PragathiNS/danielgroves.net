Server Observer

Server Observer is the outcome of my final year project at University where I studied Web Applications Development. The project monitors linux systems, submitting hardware level detail to a remote service with matches and triggers alerts against the system.

The system system as been written in Python, with the client-side softeare compiled into a binary while the web-service has been developed on top of the Django framework. The system has been built to scale, making use of Redis based queue servers and fast web servers. Each componenet can be scaled upwards as well as sideways with relative ease due to the use of Ansible based server provisioners.

The system is currently under-going a private Beta phase, and is under active development. To find out more visit serverapp.io.

Writhlington Orchid Project

I've been working with the Writhlington Orchid Project for almost as long as I've worked with the web. The Writhington Orchid Project required a website which they could use to keep those who support the project up to date with the latest news and information.

Further, as one of the largest micro-propagation labs in the world, they needed a way to provide people with access to the vast amounts of information about the orchids they have compiled. In order to do this they needed a site that could be easily updated by any member of the project without any training.

On top of this the project does several international exploration and educational trips every year. As a result they needed to be able to update the site from low-bandwidth connections on mobile devices. To provide all of this the site makes use of the WordPress platform along with some of the latest HTML5 and CSS3 techniques to ensure it is accessible on mobile devices.

Thanks to the WordPress platform the team can update the site via email and mobile applications from anywhere in the world and schedule posts to go live during UK hours, when the majority of the users are on-line.

CoCamp

CoCamp was a group based coursework assignment based around CoCamp as a case study. The brief was to develop a system where the CoCamp event could be co-ordinated. Organisers could sign up for an account, add their event to the system and order all the required catering by adding their recipes and using those of other members.

This project was completed in collaboration with Michael Wade and Nathan Davies.

A demo of this project will be available soon.
