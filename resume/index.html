Skills

As a developer I like to keep work agile. I use Git for version control, making it easy to collaborate with large teams of other developers. I regularly write unit tests for applications, often working to the test driven development methodology. This allows for easy automated testing of software to ensure that changes in the future to not break anything. I will often run these on a continuous integration server. When developing larger application I prefer to use a web framework, as this provides the scaffolding for the project and saves on development time.

When designing I work directly in the browser. This allows designs to be developed quickly whilst showing the full dimension of of a design from desktop all the way through to mobile, something missed by graphical representations. In the past I have worked in Adobde Fireworks and Photoshop to produce site mockups and Adobe Dreamweaver to produce finished products, although I prefer to hand-code sites as it provides more flexibility and more options for incorporating new technologies.

When FTP is required I use Transmit on Macs, or FileZilla on any other platform, while coding my editor of choice is Sublime Text or TextMate, although I am open to almost anything. During my first year at university I used Flash in order to produce a game.
